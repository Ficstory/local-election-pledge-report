# 후보자 검색 화면명세서

작성일: 2026-06-10

## 1. 문서 목적

이 문서는 `후보자 검색` 화면을 검색 중심의 제품 화면으로 정의하기 위한 화면명세서다.

이 화면은 후보자명, 지역, 선거유형으로 후보자를 검색하고, 검색 결과에서 후보 기본정보와 주요 공약을 확인한 뒤 후보 상세와 원문 선거자료로 이동하는 화면이다.

이 화면에서는 아래 항목을 후보 카드, 검색 필터, 비교표의 지표로 사용하지 않는다.

- 전과 공개자료
- 공약 수
- 성향 추정
- 정책축
- 근거 키워드
- 5대공약 보유 여부
- 선거공보 보유 여부
- 카드/비교표의 선거유형 반복 표기

제외 이유:

- 전과 공개자료, 5대공약, 선거공보는 모든 후보에서 확인 가능한 기본 자료 접근 영역이므로 비교 지표가 아니다.
- 공약 수는 후보자 공약 전수 분석을 화면 지표로 삼을 단계가 아니므로 제외한다. 단, 후보별 `주요 공약 제목과 짧은 요약`은 검색 결과에서 보여준다.
- 성향 추정, 정책축, 근거 키워드는 산식과 검증 기준이 없는 파생 지표다.
- 선거유형은 검색 패널 상단에서 이미 선택하므로 결과 카드에서 반복할 필요가 없다.

## 2. 화면 요약

| 항목 | 내용 |
| --- | --- |
| 화면명 | 후보자 검색 |
| 전역 네브바 라벨 | 후보자 검색 |
| 화면 목적 | 후보자명, 지역, 선거유형으로 후보자를 검색하고 주요 공약을 확인한 뒤 후보 상세/원문 자료로 이동한다. |
| 주요 사용자 | 특정 지역/선거유형의 후보를 빠르게 찾고 원문 자료를 확인하려는 사용자 |
| 주요 과업 | 검색 조건 입력, 결과 확인, 주요 공약 확인, 후보 비교, 후보 상세 이동, 원문 자료 열람 |
| 권장 신규 URL | `/candidates/search` |
| 현재 코드 대응 | `/?election=education`의 교육감 후보 검색/비교 기능을 검색 중심으로 재구성 |
| 검색 전 결과 노출 | 하지 않음 |
| 비교 기능 위치 | 검색 결과의 보조 기능 |

## 3. 설계 원칙

1. 검색 전에는 후보 카드 목록을 보여주지 않는다.
2. 사용자가 검색어 또는 필터를 입력한 뒤에만 결과 목록을 보여준다.
3. 비교는 검색 결과에서 후보를 선택했을 때만 표시한다.
4. 후보 카드와 비교표에는 공식 데이터로 설명 가능한 기본 사실만 표시한다.
5. 후보 카드의 핵심은 `후보자명`, `지역`, `정당/후보 구분`, `당선 결과`, `주요 공약`, `원문 자료 액션`이다.
6. 자료 보유 여부는 표시하지 않는다. 원문 자료는 모든 후보에서 열람 가능한 기본 액션으로 취급한다.
7. `데이터 출처`는 전역 네브바에서 제거하고 공통 footer 또는 상세 화면의 자료 출처 영역에 둔다.
8. 네브바에는 별도 CTA 버튼을 두지 않는다.

```text
│  └─ 후보 카드 목록

```

## 6. 검색 전 화면

검색 전 화면은 후보 목록을 보여주지 않는다. 사용자가 처음 들어왔을 때는 검색 조건 입력에 집중할 수 있도록 검색 패널과 데이터 범위 요약만 보여준다.

```text
후보자 검색
후보자 이름이나 지역으로 후보자를 찾아보세요.

[교육감] [시·도지사] [시·군·구청장]

후보자 이름 또는 지역
[____________________________________] [검색]

[상세 필터]

수록 후보 697명
중앙선거관리위원회 공개데이터 기반
```

검색 전 표시 규칙:

| 요소 | 표시 여부 | 비고 |
| --- | --- | --- |
| 페이지 제목 | 표시 | `후보자 검색` |
| 설명문 | 표시 | 후보명/지역 검색을 설명 |
| 선거유형 선택 | 표시 | 교육감 / 시·도지사 / 시·군·구청장 |
| 검색창 | 표시 | 최상위 입력 요소 |
| 상세 필터 | 접힘 상태 | 기본은 접힘 |
| 데이터 범위 요약 | 표시 | 후보 수 중심 |
| 후보 카드 목록 | 숨김 | 검색 전에는 노출하지 않음 |
| 비교 트레이 | 숨김 | 후보 선택 전에는 노출하지 않음 |

## 7. 검색 패널 명세

### 7.1 필드 목록

| 필드 | UI 타입 | 기본값 | 검색 적용 | 비고 |
| --- | --- | --- | --- | --- |
| 선거유형 | segmented control | 교육감 | 검색 조건 | 교육감, 시·도지사, 시·군·구청장 |
| 검색어 | search input | 빈 값 | 검색 버튼 또는 Enter | 후보자명, 지역명 |
| 지역 | combobox/select | 전체 | 검색 조건 | 선택한 선거유형의 후보가 있는 지역만 |
| 당선 결과 | select | 전체 | 검색 조건 | 당선, 낙선, 미확인 |
| 정렬 | select | 관련도순 또는 지역순 | 결과 정렬 | 결과 영역에서 노출 |

제외하는 필터:

| 제외 항목 | 제외 이유 |
| --- | --- |
| 전과 공개자료 | 모든 후보에서 확인 가능한 기본 자료라 검색 조건으로 의미가 없음 |
| 5대공약 있음 | 모든 후보의 기본 원문 자료 액션으로 취급 |
| 선거공보 있음 | 모든 후보의 기본 원문 자료 액션으로 취급 |
| 공약 수 | 후보별 공약 전수 분석 지표로 쓰지 않음 |
| 성향 추정 | 산식과 검증 기준이 없는 파생 지표 |
| 정책축 | 후보별 정책축 분석을 카드/비교 지표로 쓰는 단계가 아님 |

### 7.2 검색어 입력

| 항목 | 명세 |
| --- | --- |
| placeholder | `후보자 이름 또는 지역` |
| 최소 길이 | 1자 |
| Enter 동작 | 검색 실행 |
| 공백 처리 | 앞뒤 공백 제거 |
| 빈 검색어 | 지역/당선 결과 필터가 있으면 검색 가능 |
| 검색 대상 | 후보자명, 지역명, 선거구명 |

### 7.3 상세 필터

상세 필터는 검색창 아래 접이식 영역으로 둔다.

```text
[상세 필터 열기]

지역
[전체 v]

당선 결과
[전체 v]
```

필터가 1개 이상 적용되면 검색 패널 하단에 적용 조건 chip을 보여준다.

```text
적용 조건: 교육감 · 서울
[서울 x] [조건 초기화]
```

## 8. 검색 실행 조건

다음 중 하나라도 있으면 검색이 실행된 상태로 본다.

| 조건 | 검색 실행 여부 |
| --- | --- |
| 검색어 입력 | 실행 |
| 지역 선택 | 실행 |
| 당선 결과 선택 | 실행 |

검색어와 필터가 모두 비어 있으면 검색 결과 목록을 숨긴다.

## 9. URL 파라미터 제안

| 파라미터 | 예시 | 의미 |
| --- | --- | --- |
| `election` | `education` | 선거유형 |
| `q` | `서울` | 검색어 |
| `region` | `서울특별시` | 지역 |
| `result` | `elected` | 당선 결과 |
| `sort` | `relevance` | 정렬 |
| `compare` | `id1,id2` | 비교 후보 |
| `page` | `2` | 페이지 |

예시:

```text
/candidates/search?election=education&q=서울&sort=relevance
```

현재 경로를 유지한다면:

```text
/?election=education&q=서울&sort=relevance
```

## 10. 결과 요약 영역

### 10.1 결과 있음

```text
"서울" 검색 결과 12명
적용 조건: 교육감 · 서울

후보 12명
[정렬: 관련도순 v]
```

### 10.2 검색어 없이 필터만 적용

```text
서울 후보 4명
적용 조건: 교육감 · 서울

후보 4명
[정렬: 지역순 v]
```

### 10.3 정렬 규칙

| 정렬 | 사용 상황 | 설명 |
| --- | --- | --- |
| 관련도순 | 검색어가 있을 때 기본 | 후보명/지역 매칭 기준 |
| 지역순 | 검색어 없이 필터만 있을 때 기본 | 지역명, 선거구 기준 |
| 이름순 | 보조 정렬 | 후보자명 가나다순 |

## 11. 후보 카드 명세

후보 카드는 검색 결과 목록의 기본 단위다. 카드만 보고 다음 판단을 할 수 있어야 한다.

1. 어느 지역 후보인가?
2. 후보 이름과 후보 구분은 무엇인가?
3. 당선 결과가 있는가?
4. 주요 공약이 무엇인가?
5. 후보 상세 또는 원문 자료로 이동할 수 있는가?
6. 비교에 추가할 후보인가?

### 11.1 카드 기본 구조

```text
[서울특별시]                              [당선]
김하늘
교육감 후보

주요 공약
1. 기초학력 진단과 맞춤 보충학습
2. 초등 돌봄 운영 시간 확대
3. 학교 안전 시설 개선

[비교에 추가] [후보 상세] [5대공약 보기] [선거공보 보기]
```

### 11.2 카드 필드

| 필드 | 필수 | 표시 위치 | 비고 |
| --- | --- | --- | --- |
| 후보자명 | 필수 | 제목 | 가장 큰 텍스트 |
| 지역명/선거구명 | 필수 | 상단 메타 | 선거유형은 반복하지 않음 |
| 정당명/후보 구분 | 권장 | 후보자명 하단 | 교육감은 `교육감 후보` |
| 당선 결과 | 권장 | 우측 배지 | 당선, 낙선, 미확인 |
| 주요 공약 | 필수 | 액션 위 | 후보별 2~3개 제목 또는 한 줄 요약 |

### 11.3 카드에서 제외할 정보

아래 정보는 후보 카드에 노출하지 않는다.

- 공약 수
- 전과 공개자료 여부
- 성향 추정
- 정책축
- 근거 키워드
- 5대공약 있음/없음
- 선거공보 있음/없음
- 선거유형 반복 표기
- API ID
- DB ID
- PDF 파일명
- 수집 시각

주요 공약 표시 규칙:

- 공약 개수를 보여주지 않는다.
- 순위나 점수처럼 보이지 않게 한다.
- 후보별 2~3개 공약 제목을 짧게 보여준다.
- 공약 제목이 길면 한 줄 말줄임 또는 두 줄 이하로 제한한다.
- 자세한 공약 원문은 `후보 상세`, `5대공약 보기`에서 확인하게 한다.

## 12. 카드 액션 명세

| 액션 | 노출 조건 | 동작 | 비고 |
| --- | --- | --- | --- |
| 비교에 추가 | 비교 미선택 후보 | 비교 후보 목록에 추가 | 최대 4명 |
| 비교 해제 | 이미 선택된 후보 | 비교 후보 목록에서 제거 | 버튼 라벨 변경 |
| 후보 상세 | 항상 | `/candidates/[id]` 이동 | 기본 상세 진입 |
| 5대공약 보기 | 항상 | 5대공약 원문 자료 열람 | 자료 보유 상태를 별도로 표시하지 않음 |
| 선거공보 보기 | 항상 | 선거공보 원문 자료 열람 | 자료 보유 상태를 별도로 표시하지 않음 |

버튼 우선순위:

1. 비교에 추가/비교 해제
2. 후보 상세
3. 5대공약 보기
4. 선거공보 보기

## 13. 후보 비교 명세

비교는 검색 결과 안에서 후보를 선택했을 때만 표시한다. 비교표는 후보 식별 정보, 당선 결과, 주요 공약을 비교한다. 원문 자료는 각 후보 열의 공통 액션으로 제공한다.

### 13.1 노출 규칙

| 선택 후보 수 | 표시 |
| --- | --- |
| 0명 | 비교 영역 숨김 |
| 1명 | 안내 패널 표시 |
| 2명 | 비교표 표시 |
| 3명 | 비교표 표시 |
| 4명 | 비교표 표시, 추가 제한 |
| 5명 추가 시도 | 추가 불가 안내 |

### 13.2 1명 선택 상태

```text
선택 후보 비교
비교할 후보를 1명 더 선택하세요.

현재 선택
서울특별시 · 김하늘

[선택 비우기]
```

### 13.3 2명 이상 선택 상태

```text
선택 후보 비교 2 / 4명
[비교 비우기]

항목              김하늘        박도윤
지역              서울특별시    서울특별시
후보 구분          교육감 후보    교육감 후보
당선 결과          당선          낙선
주요 공약          기초학력       AI교육
                  돌봄 확대      학교자율
                  학교 안전      교원지원

                  [상세] [5대공약] [선거공보]
```

### 13.4 비교 항목

| 항목 | 설명 |
| --- | --- |
| 지역 | 지역명 또는 지역+선거구 |
| 후보 구분 | 정당명 또는 교육감 후보 |
| 당선 결과 | 당선, 낙선, 미확인 |
| 주요 공약 | 후보별 주요 공약 제목 2~3개 |

비교표에서 제외하는 항목:

| 제외 항목 | 제외 이유 |
| --- | --- |
| 선거유형 | 상단에서 이미 선택한 조건 |
| 공약 수 | 현재 화면 지표로 쓰지 않음 |
| 전과 공개자료 | 모든 후보에서 확인 가능해 비교 의미 없음 |
| 5대공약 자료 | 모든 후보의 공통 액션이라 비교 의미 없음 |
| 선거공보 | 모든 후보의 공통 액션이라 비교 의미 없음 |
| 성향 추정 | 산식/검증 기준 없음 |
| 정책축/근거 키워드 | 후보별 분석 지표로 확정하지 않음 |

### 13.5 비교 제한 규칙

| 상황 | 처리 |
| --- | --- |
| 최대 4명 초과 | `최대 4명까지 비교할 수 있습니다.` 안내 |
| 비교 후보가 검색 결과 밖으로 벗어남 | 비교 영역에는 유지하되 `현재 검색 결과 밖` 표시 |
| 선거유형 변경 | 비교 후보 초기화 권장 |
| 조건 초기화 | 비교 후보도 함께 초기화 권장 |
| 후보 삭제 | 해당 열 또는 카드를 제거 |
| 비교 비우기 | 모든 비교 후보 제거 |

## 23. 수용 기준

1. 검색 전에는 후보 카드 목록이 보이지 않는다.
2. 검색어 또는 필터 적용 후에만 결과 목록이 표시된다.
3. 검색 결과 상단에 결과 수와 적용 조건이 표시된다.
4. 후보 카드에서 후보명, 지역, 후보 구분, 당선 결과, 주요 공약을 확인할 수 있다.
5. 후보 카드에 공약 수, 전과 공개자료, 성향 추정, 정책축, 자료 보유 여부, 선거유형 반복 표시가 없다.
6. 후보 카드에는 `후보 상세`, `5대공약 보기`, `선거공보 보기` 액션이 있다.
7. 후보를 1명 선택하면 비교 안내가 표시된다.
8. 후보를 2명 이상 선택하면 비교표가 표시된다.
9. 비교표에는 지역, 후보 구분, 당선 결과, 주요 공약만 표시된다.
10. 후보는 최대 4명까지 비교할 수 있다.
11. 모바일에서는 비교표가 후보별 카드로 전환된다.

## 24. 추후 확장

추후 별도 분석 로직과 검증 기준이 생긴 뒤에만 아래 기능을 추가 검토한다.

| 확장 | 전제 |
| --- | --- |
| 공약 수 표시 | 후보별 공약 데이터 범위와 누락 기준이 명확할 것 |
| 정책축 표시 | 후보별 정책축 분류 산식과 검증 기준이 있을 것 |
| 성향 추정 | 산식, 근거, 검증 기준, 표시 책임 범위가 명확할 것 |
| 검색어 공약 매칭 | 공약 본문 검색 범위와 하이라이트 기준이 명확할 것 |
