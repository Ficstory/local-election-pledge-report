# 2026-06-02 PDF 선거자료 분석 방법

## 목적

수집된 선거자료 PDF 중 분석 대상을 명확히 제한하고, 다른 작업 환경에서도 같은 방식으로 이어서 분석할 수 있도록 처리 절차와 산출물 형식을 고정한다.

이번 단계의 분석 대상은 다음 두 종류만 사용한다.

- `TOP_FIVE_PLEDGES`: 5대공약
- `ELECTION_BULLETIN`: 선거공보, 책자형/전단형 선거공보 포함

`PLEDGE_DOCUMENT`로 저장된 선거공약서 파일은 이번 1차 분석 대상에서 제외한다.

## 자료 유형 해석

NEC 정책·공약마당의 `fileinfo`와 CDN 파일명 기준으로 파일 prefix는 다음처럼 해석한다.

- `001_...`: 5대공약
- `003_...`: 선거공보
- `004_...`: 선거공약서

`004`는 별도 "추가 자료집" 라벨이 아니라 선거공약서 PDF에 붙은 파일 코드로 본다. 선거공약서는 후보별로 실제 PDF가 있는 경우도 있고, PDF URL 없이 placeholder만 있는 경우도 있다. 따라서 이번 분석에서는 제외하고, 추후 심화 분석 단계에서만 선택적으로 사용한다.

## 1차 분석 대상

### 포함

- 시장·도지사·구청장·군수 그룹
  - `sgTypecode=3`: 시·도지사선거
  - `sgTypecode=4`: 구·시·군의 장선거
- 교육감 그룹
  - `sgTypecode=11`: 교육감선거
- 다운로드 상태가 `DOWNLOADED`인 PDF
- 자료 유형이 `ELECTION_BULLETIN` 또는 `TOP_FIVE_PLEDGES`인 PDF

### 제외

- `PLEDGE_DOCUMENT`
- `SKIPPED_NO_URL`
- `FAILED`
- 원본 PDF가 로컬에 없거나 SHA-256 검증 정보가 없는 항목

## 분석 단위

분석은 후보자 1명 단위로 묶는다.

후보자 1명에 대해 가능한 입력은 다음과 같다.

- 후보자 기본정보
  - 후보자명
  - 정당명
  - 선거종류
  - 시·도
  - 선거구
- 5대공약 PDF 1개
- 선거공보 PDF 1개

두 PDF 중 하나가 없으면 있는 자료만 분석하되, 누락 상태를 결과에 명시한다.

## 전체 처리 흐름

```text
manifest JSONL
  -> 분석 대상 필터링
  -> PDF 파일 존재/해시 확인
  -> 텍스트 레이어 추출
  -> 추출 품질 판정
  -> 필요 시 페이지 이미지 렌더링 + OCR
  -> 자료 유형별 구조화
  -> 후보자 단위 통합 분석
  -> JSONL 결과 저장
  -> 진행 로그 갱신
```

## PDF 추출 전략

PDF 분석은 LLM에 PDF 원본을 바로 넣지 않는다. 먼저 텍스트와 페이지 이미지를 분리해서 안정적인 중간 산출물을 만든다.

### 1. 텍스트 레이어 추출

먼저 PDF 내부 텍스트 레이어를 추출한다.

성공 기준:

- 전체 텍스트 길이가 최소 기준 이상이다.
- 한글 비율이 충분하다.
- 페이지별 텍스트가 비어 있지 않다.
- 반복되는 깨진 문자나 인코딩 오류가 과도하지 않다.

실패 또는 보류 기준:

- 페이지 대부분이 빈 텍스트다.
- 한글이 거의 없고 특수문자만 많다.
- 공약명이나 후보자명이 전혀 잡히지 않는다.
- 표/이미지 기반 PDF라 문장 순서가 크게 깨진다.

### 2. OCR fallback

텍스트 추출 품질이 낮으면 페이지를 이미지로 렌더링한 뒤 OCR을 수행한다.

OCR 대상:

- 텍스트 레이어가 없는 이미지형 PDF
- 스캔본
- 글자가 이미지로 박힌 선거공보
- 표/인포그래픽 중심 페이지

OCR 결과는 페이지 번호와 함께 저장한다. 이후 분석 단계에서는 원본 텍스트 추출본과 OCR 결과 중 품질이 좋은 쪽을 사용한다.

### 3. 페이지 이미지 렌더링

선거공보는 시각적 구성 자체도 분석 대상이다. 따라서 텍스트 추출 성공 여부와 별개로 대표 페이지 이미지를 생성할 수 있어야 한다.

이미지 분석 후보:

- 표지
- 후보자 사진이 있는 페이지
- 핵심 공약 요약 페이지
- 비교표나 도표가 많은 페이지

## 자료 유형별 분석 항목

### 5대공약 분석

5대공약은 후보자의 핵심 정책 우선순위를 비교하기 위한 자료로 사용한다.

추출 항목:

- 공약 번호
- 공약명
- 공약 요약
- 정책 분야
- 대상 유권자 또는 대상 지역
- 문제 인식
- 실행 수단
- 예산 또는 재원 언급 여부
- 일정 또는 임기 내 목표
- 정량 목표 여부
- 구체성 점수
- 실현 가능성 메모
- 원문 근거 페이지

권장 정책 분야:

- 교통
- 주거·도시
- 복지
- 보육·교육
- 일자리·경제
- 안전
- 환경
- 문화·관광
- 행정·재정
- 농어촌
- 기타

### 선거공보 분석

선거공보는 정책뿐 아니라 후보자가 유권자에게 어떤 이미지를 전달하는지 분석하는 자료로 사용한다.

추출 항목:

- 대표 슬로건
- 후보자 소개 방식
- 주요 경력 강조점
- 공약 노출량
- 지역 현안 언급
- 정당 브랜드 노출
- 후보 개인 이미지 강조 여부
- 네거티브 또는 상대 후보 공격 여부
- 주요 사진/시각 자료 유형
- 공약보다 인물 홍보 비중이 큰지
- 원문 근거 페이지

정량화 후보:

- 전체 페이지 수
- 공약 관련 페이지 수
- 후보 경력/인물 홍보 페이지 수
- 사진·이미지 비중
- 표/도표 사용 여부
- 숫자·예산·기간 언급 수

## 후보자 단위 통합 분석

5대공약과 선거공보를 따로 분석한 뒤 후보자 단위로 합친다.

통합 분석 항목:

- 핵심 공약 분야 분포
- 5대공약과 선거공보의 공약 일치 여부
- 선거공보에서 강조한 이미지와 5대공약의 정책 방향 차이
- 지역 현안 대응 여부
- 구체적인 재원·일정·정량 목표 제시 여부
- 후보자별 정책 구체성 점수
- 후보자별 홍보 중심성 점수

## 실패 처리

후보자 단위로 실패를 격리한다. 한 후보자의 PDF 분석이 실패해도 전체 batch는 중단하지 않는다.

실패 유형:

- PDF 파일 없음
- SHA-256 불일치
- 텍스트 추출 실패
- OCR 실패
- LLM 구조화 실패
- 결과 JSON 검증 실패

실패 시 저장할 정보:

- 후보자 ID
- 자료 ID
- 자료 유형
- 파일 경로
- 실패 단계
- 에러 메시지
- 재시도 가능 여부

## 품질 검증

분석 결과를 바로 신뢰하지 않고, 샘플 검수 단계를 둔다.

검수 기준:

- 5대공약이 실제로 5개로 분리되었는가
- 공약명이 PDF 원문과 맞는가
- 예산/일정/재원 언급을 과장해서 추론하지 않았는가
- 선거공보의 슬로건과 후보 이미지 설명이 원문 페이지 근거를 갖는가
- 없는 내용을 LLM이 만들어내지 않았는가

샘플 검수 권장:

- 선거유형별 최소 5명
- 정당별 최소 3명
- OCR fallback 후보 최소 5명
- 대용량 선거공보 후보 최소 5명

## 1차 실행 순서

1. manifest에서 `ELECTION_BULLETIN`, `TOP_FIVE_PLEDGES`만 필터링한다.
2. 후보자별로 두 자료를 묶는 작업 큐를 만든다.
3. 20명 정도 smoke test를 돌려 텍스트 추출 품질을 측정한다.
4. OCR 필요 비율을 기록한다.
5. JSONL 결과 스키마를 확정한다.
6. 전체 후보 batch를 실행한다.
7. 실패 후보만 재시도한다.
8. 분석 결과를 DB에 반영할지 별도 JSONL로 유지할지 결정한다.

## 현재 결론

- 1차 분석은 `5대공약`과 `선거공보`만 사용한다.
- `004` 선거공약서는 후보별 편차가 크므로 이번 1차 비교 분석에서는 제외한다.
- PDF 원본을 LLM에 바로 넣지 않고, 텍스트 추출/OCR/구조화 단계를 거친다.
- 후보자 1명 단위 JSONL 결과를 만들고, 진행 로그로 재시작 지점을 남긴다.
